## Integration Gaps (from Integration Checker)

### Gap 1: vercel.json cron schedule missing (REGRESSION)
- **File:** `/vercel.json`
- **Current state:** `{}`
- **Expected:** `{ "crons": [{ "path": "/api/notifications/dispatch", "schedule": "*/15 * * * *" }] }`
- **Impact:** NOTIF-02, NOTIF-03 — reminders and surveys will never fire
- **Fix:** Add crons entry to vercel.json (1 line)

### Gap 2: check_and_create_booking RPC missing user_id/channel parameters
- **File:** `supabase/migrations/00010_bookings.sql` (RPC), `lib/booking/slot-checker.ts`, `lib/booking/state-machine.ts`
- **Current state:** RPC signature has no `p_user_id`/`p_channel`. State machine receives `userId` and `channel` from chat endpoint but never passes them to `checkAndCreateBooking`. All chatbot bookings have `user_id=NULL`, `channel=NULL`.
- **Impact:** NOTIF-01, NOTIF-04 — notification POST to n8n includes `userId: null, channel: null`; n8n cannot route confirmation/reminder/survey to the customer's WhatsApp/Telegram
- **Fix:** Add `p_user_id text, p_channel text` to RPC + INSERT; pass from slot-checker.ts; pass from state-machine.ts

### Gap 3: n8n_outbound_webhook not seeded
- **File:** `scripts/seed-elken.mjs`
- **Current state:** Seed sets name, personality, FAQs, templates, feature_flags — but not `n8n_outbound_webhook`
- **Impact:** `dispatchNotification()` silently returns false for all notifications until manually set
- **Fix:** Add placeholder URL to seed script UPDATE block (e.g. `n8n_outbound_webhook: null` with doc comment explaining it must be set at deploy time) OR accept as operational setup item documented in deploy guide

**A. Create `scripts/smoke-test.sh`** (per D-04, D-05):

```bash
#!/usr/bin/env bash
# Elken smoke test — 9 curl calls (3 intents x 3 languages)
# Usage: bash scripts/smoke-test.sh
#
# Required env vars in .env.local (or export directly):
#   SMOKE_TEST_URL — base URL of deployed app (e.g. https://your-app.vercel.app)
#                    Falls back to VERCEL_URL if SMOKE_TEST_URL not set
#   X_API_KEY      — valid API key for the Elken bot
set -euo pipefail

# ── Parse .env.local ─────────────────────────────────────────────────────────
ENV_FILE="$(dirname "$0")/../.env.local"
if [ -f "$ENV_FILE" ]; then
  SMOKE_TEST_URL="${SMOKE_TEST_URL:-$(grep '^SMOKE_TEST_URL=' "$ENV_FILE" 2>/dev/null | cut -d= -f2- | tr -d '"' || true)}"
  SMOKE_TEST_URL="${SMOKE_TEST_URL:-$(grep '^VERCEL_URL=' "$ENV_FILE" 2>/dev/null | cut -d= -f2- | tr -d '"' || true)}"
  X_API_KEY="${X_API_KEY:-$(grep '^X_API_KEY=' "$ENV_FILE" 2>/dev/null | cut -d= -f2- | tr -d '"' || true)}"
fi

if [ -z "${SMOKE_TEST_URL:-}" ]; then
  echo "ERROR: SMOKE_TEST_URL (or VERCEL_URL) not set in .env.local or environment"
  exit 1
fi
if [ -z "${X_API_KEY:-}" ]; then
  echo "ERROR: X_API_KEY not set in .env.local or environment"
  exit 1
fi

# Strip trailing slash
SMOKE_TEST_URL="${SMOKE_TEST_URL%/}"

BOT_ID="6176aa27-ce33-4dbc-b478-407414f86cac"
PASS=0
FAIL=0

call_chat() {
  local label="$1" message="$2" lang="$3"
  printf "  %-40s ... " "$label ($lang)"
  HTTP_STATUS=$(curl -s -o /dev/null -w "%{http_code}" --max-time 30 \
    -X POST "${SMOKE_TEST_URL}/api/chat/${BOT_ID}" \
    -H "Content-Type: application/json" \
    -H "X-API-Key: ${X_API_KEY}" \
    -d "{\"message\": \"${message}\", \"userId\": \"smoke-test-${lang}\", \"channel\": \"whatsapp\", \"conversationId\": \"smoke-${lang}-$(date +%s)\"}")
  if [ "$HTTP_STATUS" = "200" ]; then
    echo "PASS ($HTTP_STATUS)"
    PASS=$((PASS + 1))
  else
    echo "FAIL ($HTTP_STATUS)"
    FAIL=$((FAIL + 1))
  fi
}

echo ""
echo "Elken Smoke Test"
echo "================"
echo "URL: ${SMOKE_TEST_URL}"
echo "Bot: ${BOT_ID}"
echo ""

echo "Product Enquiries:"
call_chat "product_enquiry" "Tell me about Elken skincare products" "en"
call_chat "product_enquiry" "Ceritakan tentang produk penjagaan kulit Elken" "bm"
call_chat "product_enquiry" "介绍一下Elken护肤产品" "zh"

echo ""
echo "Health Concerns:"
call_chat "health_concern" "I have back pain, what products help?" "en"
call_chat "health_concern" "Saya ada sakit belakang, produk apa yang membantu?" "bm"
call_chat "health_concern" "我有背痛，有什么产品可以帮助？" "zh"

echo ""
echo "Booking Intent:"
call_chat "booking_intent" "I want to book a GenQi session" "en"
call_chat "booking_intent" "Saya ingin menempah sesi GenQi" "bm"
call_chat "booking_intent" "我想预约GenQi疗程" "zh"

echo ""
echo "────────────────────────────────────"
echo "Results: ${PASS} passed, ${FAIL} failed (of 9)"
if [ "$FAIL" -gt 0 ]; then
  echo "SMOKE TEST FAILED"
  exit 1
else
  echo "ALL TESTS PASSED"
fi
```

Make it executable: `chmod +x scripts/smoke-test.sh`

**Title:** `# n8n Integration Setup Guide`

**Section 1: Prerequisites**
- n8n instance (self-hosted or n8n.cloud) running and accessible
- WhatsApp Business API account (via 360dialog, Twilio, or official CloudAPI) OR Telegram Bot Token
- API key generated from the bot's admin dashboard (Dashboard > Bots > [Bot] > Keys tab > Generate Key)
- Note the following values:
  - Webhook URL: `https://your-vercel-url/api/chat/6176aa27-ce33-4dbc-b478-407414f86cac` (replace with your actual Vercel URL and bot ID)
  - API Key: the full key shown once during generation (format: `ethan_live_xxxxxxxxxxxxxxxx`)

**Section 2: WhatsApp Workflow Setup**
Step-by-step:
1. Create a new workflow in n8n
2. Add a **WhatsApp Trigger** node (or **Webhook** node if using a third-party WhatsApp provider) — this receives incoming messages
3. Add an **HTTP Request** node connected to the trigger:
   - Method: `POST`
   - URL: `https://your-vercel-url/api/chat/{botId}`
   - Authentication: None (API key goes in headers)
   - Headers: Add `X-API-Key` with value `{{ $credentials.apiKey }}` or paste the key directly
   - Headers: Add `Content-Type` with value `application/json`
   - Body Content Type: JSON
   - Body (JSON):
     ```json
     {
       "message": "{{ $json.message.text }}",
       "userId": "{{ $json.message.from.id }}",
       "channel": "whatsapp",
       "conversationId": "{{ $json.message.chat.id }}"
     }
     ```
   - Note: Adjust the expression paths (`$json.message.text`, etc.) to match your WhatsApp trigger node's output schema. The field names (`message`, `userId`, `channel`, `conversationId`) must remain exactly as shown.
4. Add a **WhatsApp Send Message** node (or equivalent) connected to the HTTP Request output:
   - Recipient: `{{ $json.message.from.id }}` (from the trigger)
   - Message: `{{ $node["HTTP Request"].json.body }}` (the response body from the chat endpoint)

**Section 3: Telegram Workflow Setup**
Same steps as WhatsApp with these differences:
- Trigger: **Telegram Trigger** node (listens for messages)
- HTTP Request body:
  ```json
  {
    "message": "{{ $json.message.text }}",
    "userId": "{{ $json.message.from.id }}",
    "channel": "telegram",
    "conversationId": "{{ $json.message.chat.id }}"
  }
  ```
- Response: Use **Telegram Send Message** node

**Section 4: Handling Streaming Responses**
- The chat endpoint returns a streaming response (text/event-stream)
- n8n's HTTP Request node automatically waits for the stream to complete and returns the full text
- No special streaming configuration is needed in n8n
- If using n8n Cloud, response timeout may need increasing for longer responses (Settings > Execution Timeout)

**Section 5: Testing**
1. In n8n, click "Execute Workflow" with the workflow active
2. Send a test message from WhatsApp/Telegram to your bot number
3. Verify the HTTP Request node output shows HTTP 200 and contains the bot's response text
4. Verify the response appears back in your WhatsApp/Telegram chat

**Section 6: Troubleshooting**
- HTTP 401: API key is invalid or revoked. Generate a new key from the admin dashboard.
- HTTP 404: Bot ID in the URL is incorrect. Check the bot ID from the admin dashboard.
- Timeout/no response: Check that the Vercel deployment is running. Test with `curl` first (see `scripts/smoke-test.sh`).
- Empty response: The bot may not have knowledge base documents uploaded yet. Upload documents via the admin dashboard.

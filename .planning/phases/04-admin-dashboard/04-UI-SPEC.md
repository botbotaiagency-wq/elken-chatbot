## Spacing Scale

Declared values (multiples of 4 only). Phase 4 uses Tailwind default spacing — no custom tokens.

| Token | Value | Tailwind Class | Usage |
|-------|-------|----------------|-------|
| xs | 4px | `p-1` / `gap-1` | Icon gaps, badge internal padding |
| sm | 8px | `p-2` / `gap-2` | Compact element spacing, table cell padding |
| md | 16px | `p-4` / `gap-4` | Default card content padding, form field gaps |
| lg | 24px | `p-6` / `gap-6` | Card padding, section spacing |
| xl | 32px | `p-8` / `gap-8` | Page-level layout gaps |
| 2xl | 48px | `py-12` | Major section breaks |
| 3xl | 64px | `py-16` | Not used in Phase 4 dashboard pages |

Exceptions:
- Chat bubble padding in testing console: `px-4 py-2` (16px / 8px asymmetric — matches WhatsApp density feel)
- Tab nav links in bot detail layout: `px-4 py-2` with `-mb-px` for border-bottom underline alignment
- Inline confirm row: `px-2 py-3` — intentionally compact to stay within table row rhythm

Source: Tailwind default scale, `04-CONTEXT.md` §Specific Ideas, `04-RESEARCH.md` Pattern 2

## Typography

All sizes use Tailwind text utilities. Font: system stack inherited from shadcn/ui new-york base.

| Role | Size | Tailwind | Weight | Tailwind | Line Height | Usage |
|------|------|----------|--------|----------|-------------|-------|
| Body | 14px | `text-sm` | 400 | `font-normal` | 1.5 | Table cell content, form helper text, chat bubble text |
| Label | 14px | `text-sm` | 500 | `font-medium` | 1.4 | Form field labels, column headers, tab nav links |
| Heading | 16px | `text-base` | 600 | `font-semibold` | 1.25 | Card titles (CardTitle), section headings |
| Display | 20px | `text-xl` | 600 | `font-semibold` | 1.2 | Page headings (e.g., "Bot Configuration", "Testing Console") |

**Weights in use: 2** — regular (400) for content, semibold (600) for headings. Medium (500) used only for interactive labels (tabs, table headers).

**Muted text:** `text-sm text-muted-foreground` — used for answer previews in FAQ table, metadata in debug panel, empty state body copy.

Source: shadcn/ui new-york defaults, `04-CONTEXT.md` §Visual density notes, established Phase 3 patterns

## Color

Uses shadcn/ui neutral CSS variables from `app/globals.css`. All values are CSS custom properties — do not hardcode hex.

| Role | CSS Variable | Light Mode Resolved | Usage |
|------|-------------|---------------------|-------|
| Dominant (60%) | `--background` | `hsl(0 0% 100%)` — white | Page background, card background |
| Secondary (30%) | `--secondary` / `--muted` | `hsl(0 0% 96.1%)` — near-white gray | Sidebar, table row hover, tab nav background, inline confirm row background |
| Accent (10%) | `--primary` | `hsl(0 0% 9%)` — near-black | Active tab underline border, Save button, Send button in testing console, "Configure" link hover |
| Destructive | `--destructive` | `hsl(0 84.2% 60.2%)` — red | Delete FAQ button, inline confirm destructive text, revoke key button (Phase 3 pattern) |

**Accent reserved for specifically:**
1. Active tab underline (`border-primary`) in bot detail layout nav
2. Primary action buttons: "Save changes", "Send" (testing console), "Add FAQ", "Add Template"
3. "Configure" bot link in bots list on hover/focus

**Never use accent for:** Inactive tab labels, helper text, table borders, card borders.

**Badge colors for language tags:**
- EN: `bg-secondary text-secondary-foreground` (neutral, matches shadcn Badge default)
- BM: `bg-secondary text-secondary-foreground`
- ZH: `bg-secondary text-secondary-foreground`

All language badges use the same neutral color — differentiation is by text label only, not color.

**Status badge for bots list:**
- Active: `bg-green-100 text-green-800` (Tailwind semantic green — exception to neutral palette, justified by status indicator convention)
- Inactive: `bg-secondary text-muted-foreground`

Source: `app/globals.css` CSS variable values, `04-CONTEXT.md` §Specific Ideas, Phase 3 established patterns

## Component-Level Contracts

### Bots List Page (`/bots`)

- Layout: vertical list of Cards, one per bot
- Each Card: bot name (`text-base font-semibold`), status badge, "Configure" button linking to `/bots/[botId]/personality`
- Empty state: Card with centered empty state copy (see Copywriting section)
- No pagination — plain list (seeded set is small per CONTEXT.md)

### Bot Detail Layout — Horizontal Tab Nav

- Rendered by `app/dashboard/bots/[botId]/layout.tsx`
- Tab links: Personality | Guardrails | FAQs | Templates | Testing | API Keys | Integrations
- Active tab: `border-b-2 border-primary text-primary font-medium`
- Inactive tab: `border-b-2 border-transparent text-muted-foreground hover:text-foreground`
- Tab bar: `flex gap-1 border-b` flush to the page top before Card content
- Tab link size: `px-4 py-2 text-sm font-medium`

### Personality Form

- Container: single Card with CardHeader ("Personality") + CardContent
- Fields stacked vertically with `space-y-4`:
  1. Bot Name — Input, full width, Label above
  2. Greeting (EN) — `<textarea>` 3 rows, full width, Label above
  3. Greeting (BM) — `<textarea>` 3 rows, full width, Label above
  4. Greeting (ZH) — `<textarea>` 3 rows, full width, Label above
  5. Tone — native `<select>` (Professional / Friendly / Formal), styled: `w-full rounded-md border border-input bg-background px-3 py-2 text-sm`
  6. Fallback Message — `<textarea>` 3 rows, full width, Label above
- Save button: `<Button>` at bottom of CardContent, variant default (primary), label "Save changes", disabled state "Saving..." when in flight

### Guardrails Form

- Container: single Card with CardHeader ("Guardrails") + CardContent
- Fields stacked vertically with `space-y-4`:
  1. Blocked Topic Keywords — `<textarea>` 4 rows, helper text below: "One keyword per line. Messages matching these will trigger the refuse message."
  2. Custom Refuse Message — `<textarea>` 3 rows
  3. Mandatory Disclaimer Text — `<textarea>` 3 rows, helper text: "Appended to every bot response."
  4. Max Response Length — Input type="number", Label, helper text: "Characters. Recommended: 500–1500."
  5. Off-Topic Deflection Message — `<textarea>` 3 rows
- Save button: same pattern as Personality

### FAQ List + Modal

- Filter bar: `flex items-center gap-4 mb-4` containing a Label ("Language:") and native `<select>` (All | EN | BM | ZH)
- Table: full width, `<table className="w-full text-sm">`, columns: Question | Language | Answer Preview | Actions
- Table header: `text-xs font-medium text-muted-foreground uppercase tracking-wide`
- Language badge: shadcn Badge, default variant
- Answer preview: max 80 characters, truncated with `…`, `text-muted-foreground`
- Actions column: "Edit" Button (variant ghost, size sm) + "Delete" Button (variant ghost, size sm, text-destructive)
- Delete: inline confirm row expands below the target row (see RESEARCH.md Pattern 3)
- Add FAQ button: above table, right-aligned, "Add FAQ" label, Button default variant
- Create/Edit Dialog:
  - DialogHeader: "Add FAQ" or "Edit FAQ"
  - Fields: Question (`<textarea>` 3 rows), Answer (`<textarea>` 5 rows), Language (native `<select>`: EN / BM / ZH)
  - DialogFooter: Cancel (Button ghost) + Save (Button default, "Save FAQ")

### Response Templates List + Modal

- Filter bar: same pattern as FAQ filter
- Table: columns: Intent | EN | BM | ZH | Actions
  - Intent displayed as code-style label: `font-mono text-xs bg-muted px-1.5 py-0.5 rounded`
  - Language variant cells: checkmark (✓ as text, `text-green-600`) if variant exists, dash (`—`, `text-muted-foreground`) if missing
- Edit action only (no delete) — "Edit" Button, variant ghost, size sm
- Edit Dialog:
  - DialogHeader: "Edit Template: {intent_key}"
  - Fields: EN textarea (4 rows), BM textarea (4 rows), ZH textarea (4 rows), stacked vertically with labels
  - DialogFooter: Cancel + "Save Template" (Button default)
- No empty state needed (templates are seeded; always 5 rows present)

### Testing Console (`/bots/[botId]/testing`)

- Layout: full-height flex column; toolbar at top, chat window in middle (flex-grow, overflow-y-auto), input bar at bottom
- Toolbar: `flex items-center gap-4 px-4 py-2 border-b bg-background`
  - Label: "Language:" + native `<select>` (Auto | EN | BM | ZH)
  - Reset button: Button variant outline, size sm, icon `RotateCcw` from lucide-react, label "Reset"
- Chat window: `flex flex-col gap-3 p-4`
  - User message: `self-end bg-primary text-primary-foreground rounded-2xl rounded-br-sm px-4 py-2 max-w-[70%] text-sm`
  - Bot message: `self-start bg-secondary text-secondary-foreground rounded-2xl rounded-bl-sm px-4 py-2 max-w-[70%] text-sm`
  - Typing indicator: three animated dots inside a bot-style bubble, shown while streaming has not started
  - Streaming: tokens append to bot bubble content in real time
- Debug panel (below each bot message, collapsed by default):
  - Toggle: `text-xs text-muted-foreground underline cursor-pointer mt-1` — "View details" / "Hide details"
  - Expanded: `border-t mt-2 pt-2 space-y-1 text-xs text-muted-foreground`
  - Shows: Intent, RAG Found (true/false as badge), Latency ({N}ms), Source Chunks (doc name + similarity score + content preview of first 120 characters)
  - Source chunk list: `space-y-1`, each chunk: `rounded bg-muted px-2 py-1`
- Input bar: `flex gap-2 p-4 border-t`
  - Input type="text", placeholder "Type a test message…", flex-grow
  - Send button: Button default, icon `Send` from lucide-react, disabled when input empty or streaming in progress

Source: `04-CONTEXT.md` §Testing Console Layout, §Specific Ideas, `04-RESEARCH.md` Pattern 4

## Interaction Contracts

### Form State

| State | Visual |
|-------|--------|
| Default | Fields show current saved value (loaded on mount via GET) |
| Dirty (edited, unsaved) | No visual dirty indicator — no auto-save, no draft/publish |
| Saving | Save button disabled, label changes to "Saving…" |
| Save success | Button returns to "Save changes", success toast fires |
| Save error | Button returns to "Save changes", error toast fires |

### Modal Open / Close

- Dialog opens on "Add FAQ", "Edit" buttons
- Dialog closes on: Cancel button, Escape key, click outside (shadcn Dialog default behavior)
- On successful save: Dialog closes, list refetches, success toast fires
- On error: Dialog stays open, error toast fires

### Inline Confirm (Delete FAQ)

1. Admin clicks "Delete" — inline confirm row inserts below the target row
2. Only one confirm row open at a time (clicking Delete on another row replaces the current confirm)
3. "Yes, delete" triggers DELETE request, row removed from list on success, success toast fires
4. "Cancel" collapses the confirm row, no change

### Testing Console Streaming

1. Admin types message, presses Enter or clicks Send
2. User bubble appears immediately (optimistic)
3. Bot bubble appears with typing indicator ("…") within 200ms
4. First token replaces indicator — tokens stream into bubble in real time
5. Stream completes — latency stamp calculated
6. Second fetch fires for source chunks from debug endpoint
7. "View details" toggle becomes available below the bot bubble

### Language Override (Testing Console)

- Default: "Auto" — language detection runs on each message
- When a language is selected (EN/BM/ZH): `language_override` field included in every POST body until reset or changed
- Override persists across messages until changed by admin or conversation is reset
- Reset clears: message history, conversationId, language override (dropdown returns to Auto)

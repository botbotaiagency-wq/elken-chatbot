**Migration 00009_bot_config.sql:**
Create migration adding personality and guardrails columns to the `bots` table:

```sql
-- Personality fields
ALTER TABLE public.bots
  ADD COLUMN IF NOT EXISTS greeting_en        text,
  ADD COLUMN IF NOT EXISTS greeting_bm        text,
  ADD COLUMN IF NOT EXISTS greeting_zh        text,
  ADD COLUMN IF NOT EXISTS tone               text DEFAULT 'Professional'
                                              CHECK (tone IN ('Professional', 'Friendly', 'Formal')),
  ADD COLUMN IF NOT EXISTS fallback_message   text;

-- Guardrails fields
ALTER TABLE public.bots
  ADD COLUMN IF NOT EXISTS blocked_keywords       text,
  ADD COLUMN IF NOT EXISTS refuse_message         text,
  ADD COLUMN IF NOT EXISTS disclaimer_text        text,
  ADD COLUMN IF NOT EXISTS max_response_length    integer DEFAULT 1000,
  ADD COLUMN IF NOT EXISTS off_topic_message      text;
```

No new tables — faqs and response_templates already exist with correct structure.

**app/api/config/[botId]/personality/route.ts:**
- Export `GET` and `PATCH`
- `params` typed as `Promise<{ botId: string }>` with `const { botId } = await params`
- `GET`: `createServiceClient()` -> `.from('bots').select('name, greeting_en, greeting_bm, greeting_zh, tone, fallback_message').eq('id', botId).single()` -> return JSON data or 404
- `PATCH`: parse body `{ name, greeting_en, greeting_bm, greeting_zh, tone, fallback_message }` -> `.from('bots').update(...)` -> return `{ ok: true }` or 500
- Validate `tone` is one of `'Professional' | 'Friendly' | 'Formal'` before updating

**app/api/config/[botId]/guardrails/route.ts:**
- Export `GET` and `PATCH`
- Same params pattern as personality
- `GET`: select `blocked_keywords, refuse_message, disclaimer_text, max_response_length, off_topic_message` from bots where id = botId
- `PATCH`: parse body with those same fields -> `.from('bots').update(...)` -> return `{ ok: true }`
- Validate `max_response_length` is a positive integer if provided

**app/api/config/[botId]/faqs/route.ts:**
- Export `GET`, `POST`, `PATCH`, `DELETE`
- `GET`: `.from('faqs').select('*').eq('bot_id', botId).order('created_at', { ascending: false })` -> return `{ faqs: data }`
  - Support optional `?language=en|bm|zh` query param to filter
- `POST`: parse body `{ question, answer, language }` -> `.from('faqs').insert({ bot_id: botId, question, answer, language })` -> return created faq row with 201 status
  - Validate language is one of 'en', 'bm', 'zh'
  - Validate question and answer are non-empty strings
- `PATCH`: parse body `{ faqId, question, answer, language }` -> `.from('faqs').update({ question, answer, language }).eq('id', faqId).eq('bot_id', botId)` -> return `{ ok: true }`
  - The `.eq('bot_id', botId)` ensures bot-scoping (cannot edit another bot's FAQ)
  - Validate same fields as POST
- `DELETE`: parse body `{ faqId }` -> `.from('faqs').delete().eq('id', faqId).eq('bot_id', botId)` -> return `{ ok: true }`
  - The `.eq('bot_id', botId)` ensures bot-scoping (cannot delete another bot's FAQ)

**app/api/config/[botId]/templates/route.ts:**
- Export `GET` and `PATCH` (no DELETE — templates are tied to fixed intents)
- `GET`: `.from('response_templates').select('*').eq('bot_id', botId).order('intent_key')` -> return `{ templates: data }`
  - Support optional `?language=en|bm|zh` query param
- `PATCH`: parse body `{ intent_key, language, content }` -> use `.upsert({ bot_id: botId, intent_key, language, content }, { onConflict: 'bot_id,intent_key,language' })` -> return `{ ok: true }`
  - This handles both insert (new language variant) and update (existing variant)
  - Validate intent_key is one of: 'no_product_found', 'slot_full', 'booking_confirmed', 'reminder_24h', 'post_survey'
  - Validate language is one of 'en', 'bm', 'zh'

**app/api/bots/route.ts:**
- Export `GET`
- Create server-side Supabase client using `createClient` from `@/lib/supabase/server` (NOT service client) to get authenticated user session
- `const { data: { user } } = await supabase.auth.getUser()` -> if no user, return 401
- Lookup user's profile: `createServiceClient()` -> `.from('profiles').select('tenant_id, role').eq('id', user.id).single()`
- If role === 'super_admin': query ALL bots (no tenant filter)
- If role === 'tenant_admin': query bots WHERE `tenant_id = profile.tenant_id`
- Select: `id, name, feature_flags, created_at, tenant_id`
- Return `{ bots: data }`

**app/api/config/[botId]/test-chat/route.ts — Internal test chat endpoint (bypasses API key):**

This route is a near-copy of `app/api/chat/[botId]/route.ts` BUT without any API key validation. It exists so the testing console can call the RAG pipeline without needing an API key. It should be accessible only from the authenticated dashboard.

```typescript
import crypto from 'crypto'
import Anthropic from '@anthropic-ai/sdk'
import { detectIntentAndLanguage } from '@/lib/rag/detect'
import { retrieveContext } from '@/lib/rag/retrieve'
import { buildSystemPrompt } from '@/lib/rag/prompt'
import type { BotConfig } from '@/lib/rag/prompt'
import { logMessage, getOrCreateConversation } from '@/lib/rag/logger'
import { createServiceClient } from '@/lib/supabase/service'
import type { Language } from '@/types/database'

export const maxDuration = 60
```

Export `POST` function:
1. `const { botId } = await params` (params typed as `Promise<{ botId: string }>`)
2. `const supabase = createServiceClient()`
3. Fetch bot with ALL config columns (same expanded SELECT as updated chat route): `id, name, greeting_en, greeting_bm, greeting_zh, tone, fallback_message, blocked_keywords, refuse_message, disclaimer_text, max_response_length, off_topic_message`
4. If bot not found, return 404
5. **NO API key validation** — this is the key difference from the chat route
6. Parse body: `{ message, userId, channel, conversationId: inputConversationId, language_override }` — same as chat route
7. Validate required fields (message, userId, channel). Default userId to `'admin-test'` if not provided. Default channel to `'web'`.
8. Get or create conversation: `const conversationId = await getOrCreateConversation(botId, userId ?? 'admin-test', channel ?? 'web', inputConversationId)`
9. Log user message (same as chat route)
10. Detect intent and language: `const detection = await detectIntentAndLanguage(message)`
11. Apply language_override: `if (language_override && ['en', 'bm', 'zh'].includes(language_override)) { detection.language = language_override as Language }`
12. Retrieve context: `const retrieval = await retrieveContext(message, botId, detection.intent)`
13. Build system prompt with botConfig (same as updated chat route)
14. Build sourceChunks array (same as chat route)
15. Stream Claude response (same as chat route — Anthropic SDK, ReadableStream, TextEncoder)
16. Log assistant message after stream completes (same as chat route)
17. Return Response with same headers: `X-Conversation-Id`, `X-Intent`, `X-Language`, `X-Rag-Found`

**IMPORTANT:** This is intentionally duplicated logic (not a shared function) because the chat route has API key validation woven through it. The duplication is acceptable for two routes; extracting shared logic would be a premature refactor.

**app/api/config/[botId]/debug/route.ts — Debug endpoint for source chunks:**

Export `GET` function:
1. `const { botId } = await params`
2. Parse `conversationId` from URL search params: `const { searchParams } = new URL(req.url)` -> `const conversationId = searchParams.get('conversationId')`
3. If no conversationId, return 400: `{ error: 'Missing conversationId query param' }`
4. `const supabase = createServiceClient()`
5. Verify conversation belongs to this bot: `const { data: conv } = await supabase.from('conversations').select('id').eq('id', conversationId).eq('bot_id', botId).single()`
6. If no conv, return 404: `{ error: 'Conversation not found for this bot' }`
7. Fetch the LAST assistant message in this conversation: `const { data: msg } = await supabase.from('messages').select('id, intent, source_chunks, rag_found, latency_ms').eq('conversation_id', conversationId).eq('role', 'assistant').order('created_at', { ascending: false }).limit(1).single()`
8. If no msg or no source_chunks, return: `{ intent: msg?.intent, rag_found: msg?.rag_found, latency_ms: msg?.latency_ms, source_chunks: [] }`
9. Parse source_chunks (jsonb array of `{ chunk_id, similarity }`). For each chunk_id, resolve it:
   - Query chunks table: `select('id, content, document_id')` where id IN chunk_ids AND bot_id = botId
   - Query documents table for the document_ids to get filenames
   - Or do a single query with a join: for each chunk, get the chunk content (first 120 chars as preview) and the parent document filename

   Implementation:
   ```typescript
   const chunkIds = (msg.source_chunks as Array<{chunk_id: string, similarity: number}>).map(c => c.chunk_id)
   const { data: chunks } = await supabase
     .from('chunks')
     .select('id, content, document_id, documents!inner(filename)')
     .in('id', chunkIds)
     .eq('bot_id', botId)
   ```

   Note: If the join syntax `documents!inner(filename)` doesn't work with the Supabase JS client, fall back to two queries:
   - First: get chunks by IDs
   - Second: get documents by document_ids
   - Map them together

10. Build resolved source_chunks array:
    ```typescript
    const resolved = (msg.source_chunks as Array<{chunk_id: string, similarity: number}>).map(sc => {
      const chunk = chunks?.find(c => c.id === sc.chunk_id)
      return {
        chunk_id: sc.chunk_id,
        similarity: sc.similarity,
        content_preview: chunk?.content?.slice(0, 120) ?? '',
        document_name: (chunk as any)?.documents?.filename ?? 'Unknown',
      }
    })
    ```

11. Return: `{ intent: msg.intent, rag_found: msg.rag_found, latency_ms: msg.latency_ms, source_chunks: resolved }`

**sendMessage async function:**
1. If inputText is empty or streaming, return
2. `const text = inputText; setInputText(''); setStreaming(true)`
3. Generate user message ID: `const userMsgId = crypto.randomUUID()`
4. Add user message to state: `setMessages(prev => [...prev, { id: userMsgId, role: 'user', content: text }])`
5. Generate bot message ID: `const botMsgId = crypto.randomUUID()`
6. Add pending bot message: `setMessages(prev => [...prev, { id: botMsgId, role: 'assistant', content: '', pending: true }])`
7. `const startTime = Date.now()`
8. Try/catch block:
   ```typescript
   const res = await fetch(`/api/config/${botId}/test-chat`, {
     method: 'POST',
     headers: { 'Content-Type': 'application/json' },
     body: JSON.stringify({
       message: text,
       userId: 'admin-test',
       channel: 'web',
       conversationId: conversationId || undefined,
       language_override: languageOverride !== 'auto' ? languageOverride : undefined,
     }),
   })
   ```
9. If `!res.ok`, throw error
10. Read metadata from headers immediately:
    ```typescript
    const intent = res.headers.get('X-Intent')
    const ragFound = res.headers.get('X-Rag-Found') === 'true'
    const newConversationId = res.headers.get('X-Conversation-Id')
    if (newConversationId) setConversationId(newConversationId)
    ```
11. Stream the body:
    ```typescript
    const reader = res.body!.getReader()
    const decoder = new TextDecoder()
    let fullText = ''

    while (true) {
      const { done, value } = await reader.read()
      if (done) break
      const chunk = decoder.decode(value, { stream: true })
      fullText += chunk
      setMessages(prev => prev.map(m =>
        m.id === botMsgId ? { ...m, content: fullText, pending: false } : m
      ))
    }
    ```
12. Calculate latency: `const latencyMs = Date.now() - startTime`
13. After stream completes, fetch debug data:
    ```typescript
    let sourceChunks: SourceChunk[] = []
    if (newConversationId) {
      try {
        const debugRes = await fetch(`/api/config/${botId}/debug?conversationId=${newConversationId}`)
        if (debugRes.ok) {
          const debugData = await debugRes.json()
          sourceChunks = debugData.source_chunks ?? []
        }
      } catch { /* silent */ }
    }
    ```
14. Update bot message with metadata:
    ```typescript
    setMessages(prev => prev.map(m =>
      m.id === botMsgId ? { ...m, intent, ragFound, latencyMs, sourceChunks } : m
    ))
    ```
15. Catch block: on error, update bot message as error:
    ```typescript
    setMessages(prev => prev.map(m =>
      m.id === botMsgId ? { ...m, content: 'Something went wrong. The bot did not respond. Check the console and try again.', pending: false, error: true } : m
    ))
    ```
16. Finally: `setStreaming(false)`

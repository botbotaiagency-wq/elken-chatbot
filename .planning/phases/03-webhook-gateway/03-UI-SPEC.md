## Spacing Scale

Standard 8-point scale. All values are multiples of 4.

| Token | Value | Tailwind Class | Usage |
|-------|-------|---------------|-------|
| xs | 4px | gap-1 / p-1 | Icon gaps, badge internal padding |
| sm | 8px | gap-2 / p-2 | Compact row spacing, button icon gaps |
| md | 16px | gap-4 / p-4 | Default element spacing, card padding |
| lg | 24px | gap-6 / p-6 | Section padding, sidebar item groups |
| xl | 32px | gap-8 / p-8 | Main content padding (matches existing `p-8` in dashboard layout) |
| 2xl | 48px | gap-12 | Major section breaks between page sections |
| 3xl | 64px | gap-16 | Page-level spacing (unused in this phase) |

Exceptions: none for this phase. No touch targets — this is a desktop-only admin dashboard.

## Typography

Source: existing dashboard layout (`text-lg font-semibold` heading, standard Tailwind prose). Inherits shadcn neutral CSS variable scale.

| Role | Size | Weight | Line Height | Tailwind |
|------|------|--------|-------------|---------|
| Body | 14px | 400 (regular) | 1.5 | `text-sm` |
| Label | 14px | 500 (medium) | 1.4 | `text-sm font-medium` |
| Heading | 18px | 600 (semibold) | 1.2 | `text-lg font-semibold` |
| Display | 24px | 700 (bold) | 1.2 | `text-2xl font-bold` |

Notes:
- Body (14px) is the default for table rows, descriptions, snippet code blocks.
- Label (14px medium) is used for form field labels, key prefix badges, timestamp values.
- Heading (18px semibold) is used for page section titles and card titles — matches the existing sidebar `text-lg font-semibold` pattern.
- Display (24px bold) is used only for the page-level `<h1>` of the API Keys and Integrations pages.
- Monospace exception: the generated API key and the n8n JSON snippet use `font-mono text-sm` (14px). This is a display style within the Body role — not a fifth typography tier.

### Revoke Key Confirmation

Confirmation approach: inline destructive confirmation within the table row — no separate modal. When admin clicks "Revoke", the row expands to show a compact confirmation inline.

| Element | Copy |
|---------|------|
| Confirmation prompt | "Revoke this key? Requests using it will immediately return 401." |
| Confirm button | "Yes, revoke" |
| Cancel button | "Cancel" |

Rationale: A full Dialog for revocation is overkill and adds extra clicks for a reversible-by-regeneration action. Inline confirmation is faster and consistent with admin tool UX conventions.

## Interaction Contract

### Show-Once Key Modal

- Dialog does NOT close on outside click: `onInteractOutside={(e) => e.preventDefault()}`
- Dialog does NOT close on Escape key: `onEscapeKeyDown={(e) => e.preventDefault()}`
- Only the "I've copied this key" button dismisses the dialog
- Copy button triggers `navigator.clipboard.writeText(key)` — async Clipboard API (not deprecated `execCommand`)
- After copy: icon swaps from `Copy` to `Check` for 2000ms, then reverts
- Key displayed in a `readOnly` `Input` with `font-mono text-sm` styling
- After modal closes: the generated key variable is cleared from component state immediately

### API Key List

- Table renders with columns: Label, Key Prefix, Last Used, Created, Actions
- Key prefix displayed as `Badge` variant `outline` with monospace font: e.g. `a3f8b21c`
- "Last Used" shows "Never" when `last_used_at` is null
- "Last Used" shows relative time (e.g. "3 days ago") when populated — updated on next page load, not real-time
- Revoke flow: click "Revoke" icon → row expands inline confirmation → "Yes, revoke" sends DELETE → row removes from list optimistically → on error: row reappears with error toast
- Revoked keys do not appear in the list (query filters `revoked_at IS NULL`)

### Copy-to-Clipboard Pattern

Used in: key modal, webhook URL field, n8n snippet block.

```
Click copy button
  → navigator.clipboard.writeText(value)
  → icon: Copy → Check
  → after 2000ms: icon: Check → Copy
```

No toast notification on copy — the icon swap is the feedback signal. Toasts are reserved for async server operations (generate, revoke).

### n8n Snippet Display

- Rendered as `<pre><code>` block with Tailwind classes: `bg-muted rounded-md p-4 text-sm font-mono overflow-x-auto`
- No syntax highlighting library — inline Tailwind-styled `<pre><code>` (sufficient for two static JSON snippets; avoids ~70KB react-syntax-highlighter dependency)
- Copy button sits at the top-right corner of the code block, absolutely positioned

### Toast Notifications

Used for async server operations only (not copy feedback).

| Trigger | Toast Type | Message |
|---------|-----------|---------|
| Generate key — success | (none — modal opens instead) | — |
| Generate key — error | Destructive | "Failed to generate key. Please try again." |
| Revoke key — success | (none — row disappears) | — |
| Revoke key — error | Destructive | "Failed to revoke key. Please try again." |

Note: shadcn/ui `Sonner` (toast) is not yet installed. The executor should install it:
```bash
npx shadcn@latest add sonner
```
Or use the existing shadcn `toast` if preferred — either is acceptable. The contract requires destructive variant for error toasts.

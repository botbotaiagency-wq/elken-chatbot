**State variables:**
- `keys: Array<{ id, label, key_prefix, last_used_at, created_at }>` — fetched on mount
- `loading: boolean` — for initial fetch
- `generating: boolean` — for generate button loading state
- `labelInput: string` — controlled input for key label
- `showKeyModal: boolean` — controls show-once Dialog
- `generatedKey: string | null` — plaintext key shown once in modal
- `copied: boolean` — icon swap state (Copy -> Check for 2000ms)
- `revokingKeyId: string | null` — which key row is showing inline revoke confirmation

**On mount (useEffect):**
- Fetch `GET /api/keys/${botId}` to populate `keys` state
- Extract `botId` from URL using `useParams()` from `next/navigation`

**Generate Key flow:**
- Label input with placeholder "e.g. n8n Production" and "Generate Key" primary button (right-aligned in CardHeader)
- On click: POST `/api/keys/${botId}` with `{ label: labelInput }`
- On success: set `generatedKey` to response.key, set `showKeyModal` to true, refetch key list, clear label input
- On error: show destructive toast "Failed to generate key. Please try again." via sonner

**Show-Once Modal (Dialog):**
- `open={showKeyModal}` controlled
- `onInteractOutside={(e) => e.preventDefault()}` — prevents outside click dismiss
- `onEscapeKeyDown={(e) => e.preventDefault()}` — prevents Escape dismiss
- DialogTitle: "Your new API key"
- DialogDescription: "This key will not be shown again. Copy it now and store it securely."
- Warning text with AlertTriangle icon: "Once you close this dialog, the key cannot be retrieved."
- Read-only Input with `font-mono text-sm` displaying `generatedKey`
- Copy button beside input: `navigator.clipboard.writeText(generatedKey)` — on click, set `copied=true`, after 2000ms set `copied=false`. Show `Check` icon when copied, `Copy` icon otherwise.
- "I've copied this key" dismiss button (primary variant) — on click: set `showKeyModal=false`, set `generatedKey=null` (clear from state immediately)

**Key list table:**
- Columns: Label | Key Prefix | Last Used | Created | Actions
- Key prefix displayed in `Badge` variant `outline` with `font-mono`: e.g. `a3f8b21c`
- Last Used: show "Never" when `last_used_at` is null, relative time (e.g. "3 days ago") when populated. Use manual relative time calculation or install date-fns `formatDistanceToNow` — either is acceptable.
- Created: formatted date string
- Actions column: `Trash2` icon button (ghost variant) to trigger inline revoke confirmation

**Inline revoke confirmation:**
- When `revokingKeyId === key.id`, the row expands to show:
  - Text: "Revoke this key? Requests using it will immediately return 401."
  - "Yes, revoke" button (destructive variant)
  - "Keep key" button (ghost variant)
- On "Yes, revoke": DELETE `/api/keys/${botId}` with `{ keyId: key.id }` — optimistically remove from state, on error re-add and show destructive toast "Failed to revoke key. Please try again."
- On "Keep key": set `revokingKeyId=null`

**Empty state:**
- When `keys.length === 0` and not loading:
  - Heading: "No API keys yet"
  - Body: "Generate your first API key to start authenticating webhook requests."

**Page structure (from UI-SPEC):**
- Page heading (h1, `text-2xl font-semibold`): "API Keys"
- Page description (`text-sm text-muted-foreground`): "Manage API keys for authenticating webhook requests to this bot."
- Card wrapping the key list
- CardHeader contains "API Keys" title and the generate key form (label input + button) right-aligned

**Icons (lucide-react):** Key (page icon), Copy, Check, Trash2, AlertTriangle

**Toast setup:** Add `<Toaster />` from sonner. If dashboard layout already has it, skip. Otherwise add it to this page component or the dashboard layout.

## Spacing Scale

Standard 8-point scale in use across all prior phases. Tailwind default utility classes apply.

| Token | Value | Usage |
|-------|-------|-------|
| xs | 4px | Icon gaps (`gap-1`), inline padding |
| sm | 8px | Compact element spacing (`gap-2`, `p-2`) |
| md | 16px | Default element spacing (`gap-4`, `p-4`) |
| lg | 24px | Section padding (`space-y-6`, `p-6`) |
| xl | 32px | Layout gaps (`space-y-8`) |
| 2xl | 48px | Major section breaks |
| 3xl | 64px | Page-level spacing (not used in dashboard pages) |

Exceptions:
- Date filter toggle buttons: minimum 44px touch target height (`h-11`) for the segmented button group
- Chart containers: minimum height 200px for area/line chart, 180px for bar chart — prevents collapsed empty state
- Stat card row (booking funnel): fixed height `h-20` per card for alignment

Pattern reference: bookings/page.tsx uses `space-y-6` at page level, `p-4` inside CardContent, `gap-2` in filter bars.

### Bot Selector Gate
- When no bot is selected: render only the bot selector, no tabs, no date filter
- Copy: "Select a bot to view analytics:" (Label) + Select placeholder "Choose a bot..."
- This matches the exact gate pattern from `/dashboard/bookings/page.tsx`

### Date Filter Bar
- Visible only after a bot is selected
- Three preset buttons rendered as a segmented toggle: `Today` | `7 days` | `30 days`
- Active preset: `variant="default"` (filled, near-black background)
- Inactive preset: `variant="outline"`
- Fourth button: `Custom` — opens a Popover containing the Calendar component for date range selection
- When Custom is active: display the selected range as "Mar 1 – Mar 22" inside the button
- Default on page load: `7d` preset selected

## Chart Specifications

### Message Volume Chart (ANAL-01)
- Type: Area chart (`AreaChart` from Recharts via shadcn chart)
- X-axis: dates (day granularity for 7d/30d; hour granularity for today)
- Y-axis: message count (integer, no decimals)
- Series: one area per channel — WhatsApp (green: `hsl(142, 76%, 36%)`), Telegram (blue: `hsl(210, 100%, 56%)`). If no channel filter is applied, show combined total as a single neutral area using `--chart-1`
- Container height: 200px
- ChartTooltip: show date + count per channel

### Intent Breakdown Chart (ANAL-02)
- Type: Horizontal bar chart (`BarChart` layout="vertical")
- Y-axis labels: `browse_product`, `health_issue`, `book_session`, `faq`, `unknown` — rendered with display labels: "Browse Product", "Health Issue", "Book Session", "FAQ", "Unknown"
- X-axis: count (integer)
- Bar fill colors: chart-1 through chart-5 in order (one color per intent)
- Each bar label: show `{count} ({percentage}%)` to the right of bar
- Container height: 180px

### Booking Funnel (ANAL-11)
- Not a chart. Four `Card` boxes in a horizontal `grid grid-cols-4 gap-4` row
- Each box shows: label (12px muted), large count (28px semibold), conversion percentage below (12px muted)
- Labels: "Enquiries", "Submitted", "Confirmed", "Attended"
- Conversion % formula: count ÷ Enquiries × 100, rounded to 1 decimal
- No arrows or connectors — plain stat boxes only

## Interaction Contracts

### Loading State
- Chart areas: show a `div` with `animate-pulse bg-muted rounded-md` at the exact chart container height (200px / 180px) while data is fetching
- Table areas: show 3 skeleton rows — `div h-8 animate-pulse bg-muted rounded` per row
- Stat cards: show `div h-8 w-24 animate-pulse bg-muted rounded` in place of the number
- No spinner components. Pulse skeleton only.

### Empty State (no data for selected period)
- Chart empty state: centered inside the chart container area, not a separate card
  - Heading: 14px semibold — "No data for this period"
  - Body: 12px muted — "Try selecting a longer date range or a different bot."
- Table empty state: centered row inside the table body
  - "No records for this period." (14px muted-foreground, `text-center py-8`)
- Booking funnel empty state: all 4 boxes show "0" with "—%" — do not hide the funnel row

### CSV Export
- Button placement: top-right of each Card's `CardHeader`, alongside the card title
- Button label: "Export CSV" with `Download` lucide icon (16px, `h-4 w-4 mr-2`)
- Button variant: `outline`, size `sm`
- Export behavior: client-side generation from the already-fetched data array
- On export success: `toast.success("Report exported.")` via sonner
- Export respects the currently active date filter — the filename encodes the range: `{report-name}_{botId}_{from}_{to}.csv`
- Examples: `message-volume_abc123_2026-03-15_2026-03-22.csv`, `intent-breakdown_abc123_today.csv`

### Date Filter Interaction
- Switching presets (today / 7d / 30d): immediately re-fetches all visible tab data with new date range
- Switching tabs: does NOT reset the date filter — date selection is global and persists across tab changes
- Custom range: both start and end date must be selected before a fetch is triggered; "Custom" button shows "Select range..." until both are chosen

### Tab Switching
- Switching tabs: does NOT re-fetch data if the same date range and bot are still selected — cache the last fetch result per tab
- On bot change: clear all cached results and re-fetch the active tab

## Accessibility Contract

- All chart containers: `role="img"` with `aria-label` describing the chart (e.g. `aria-label="Message volume over the last 7 days"`)
- Export buttons: `aria-label="Export {report name} as CSV"`
- Date filter toggle buttons: use `aria-pressed` to indicate active state
- Skeleton loading containers: `aria-busy="true"` and `aria-label="Loading analytics..."`
- Tab navigation: shadcn `Tabs` component uses Radix — keyboard navigation (arrow keys) is handled by Radix, no custom wiring needed

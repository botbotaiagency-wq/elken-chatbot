## Spacing Scale

Declared values (must be multiples of 4):

| Token | Value | Usage |
|-------|-------|-------|
| xs | 4px | Icon gaps, badge inner padding |
| sm | 8px | Compact row padding, filter bar gap between controls |
| md | 16px | Default element spacing, card padding, form field gap |
| lg | 24px | Section padding, space-y-6 gap between page sections |
| xl | 32px | Layout gaps between major sections |
| 2xl | 48px | Empty state vertical padding (py-12) |
| 3xl | 64px | Not used in this phase |

Exceptions:
- Touch targets for action buttons (Confirm / Cancel / No-show): minimum 44px height to meet WCAG 2.5.5. Achieved via `h-11` (44px) on Button when rendered at row level.
- Filter bar height: 40px (`h-10`) — standard shadcn Select height.

**Source:** Inferred from existing dashboard patterns (`py-12` empty state, `space-y-6` page wrapper) — consistent with all prior phases.

## Color

All values reference shadcn CSS variables declared in `app/globals.css`.

| Role | CSS Variable | Light Mode Value | Usage |
|------|-------------|-----------------|-------|
| Dominant (60%) | `--background` | `hsl(0 0% 100%)` — white | Page background, all surfaces |
| Secondary (30%) | `--card` / `--secondary` | `hsl(0 0% 100%)` / `hsl(0 0% 96.1%)` | Cards, table rows, filter bar, Sheet panel |
| Accent (10%) | `--primary` | `hsl(0 0% 9%)` — near-black | See accent reserved-for list below |
| Destructive | `--destructive` | `hsl(0 84.2% 60.2%)` — red | Cancel booking, No-show action, inline confirm row |

Accent (`--primary`, near-black) reserved for:
- Primary CTA buttons: "Confirm Booking", "Register Walk-in", "Save Changes"
- Booking status change: "Confirm" action button per row
- Form submit buttons inside Walk-in Dialog and Sheet edit form

Destructive color (`--destructive`) reserved for:
- "Cancel Booking" action button and its inline confirmation row
- "Mark No-show" action button and its inline confirmation row
- Trash / destructive icons (`text-destructive` class)

Status badge color mapping (using shadcn Badge variants + Tailwind overrides):
- `pending` — `variant="outline"` with amber border/text: `border-amber-300 text-amber-700 bg-amber-50`
- `confirmed` — `variant="outline"` with green: `border-green-300 text-green-700 bg-green-50`
- `cancelled` — `variant="outline"` with red: `border-red-300 text-red-700 bg-red-50`
- `no-show` — `variant="outline"` with gray: `border-gray-300 text-gray-500 bg-gray-50`
- `walk-in` — `variant="outline"` with blue: `border-blue-300 text-blue-700 bg-blue-50`

**Source:** `app/globals.css` CSS variable values + existing badge usage in `api-keys/page.tsx`.

## Interaction States

| State | Element | Treatment |
|-------|---------|-----------|
| Loading | Bookings table | `py-8 text-center text-sm text-muted-foreground` — "Loading bookings..." |
| Empty | Bookings table | `py-12 text-center` — heading + body copy (see Copywriting) |
| Pending status change | Action button | `disabled` + spinner (use lucide `Loader2 animate-spin h-4 w-4`) |
| Inline confirm (cancel/no-show) | Table row expansion | `bg-destructive/5` highlight row with confirmation message + Yes/No buttons |
| Sheet loading | Sheet content | Skeleton lines (`animate-pulse bg-muted h-4 rounded`) while audit log fetches |
| Notification failed | Booking row | Badge: `border-red-300 text-red-700 bg-red-50` "Notif failed" indicator |
| Walk-in form submitting | Dialog submit button | `disabled` + "Registering..." label |
| Edit mode (Sheet) | Field values | Replace `<span>` with `<Input>` / `<Select>`, show Save + Cancel buttons |
